## Why it stands out

- real-world agricultural problem domain
- multi-role workflows instead of a single-dashboard demo
- full-stack implementation, not just static screens
- curated datasets powering explainable product features
- analytics, advisory, scanner, weather, and collaboration in one platform

## Why it still needs work

- hybrid Supabase + custom backend complexity needs continued cleanup
- some modules are still prototype-mode under the hood
- validation, testing, and monitoring are not yet complete across the whole system
- screenshots/demo media still need to be captured from the running app
